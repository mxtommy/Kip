Instructions and Upgrade Limitations

The new and old widget layout engines are fundamentally different. As a result, migrating the configuration will produce the following:

  • All KIP configuration settings will be migrated.
  • Each v2 Page layout will be migrated to a v3 Dashboard.
  • All widgets and their configurations will be migrated and visible in their corresponding v3 Dashboard.
  • Note: Widget positions, order, and sizes WILL BE LOST. Dashboards will contain small widgets that can be repositioned and resized manually.

To upgrade your configuration, please follow these steps:

  1. As a safety measure, if you are using Signal K authentication, manually back up your configuration. Make a copy of the 9.0.0.json files found on the Signal K server in the following folders:
    • ~/.signalk/applicationData/users/(username)/kip/
    • ~/.signalk/applicationData/global/kip/
  2. Click the Upgrade button below.
  3. Once the upgrade is complete, KIP will restart and display the migrated dashboards.
  4. Swipe left, or press Shift + Ctrl + Left Arrow to open the Actions menu. Tap/Click on Help and read "The Basics" and "Dashboards and Layout." We promise, it's a quick read.
  5. Reposition and resize your widgets as needed.
  6. Once satisfied, proceed to deleting the 9.0.0.json configuration files located in the previously mentioned folders. They are not needed any more.
  7. If you need support, join the Signal K #kip Discord channel.
  8. Enjoy the new features and improvements!

If you prefer not to upgrade your configuration, you MUST MANUALLY DELETE the 9.0.0.json files from the previously mentioned server folders.
